What is HTML?

The building blocks of the web

HTML stands for HyperText Markup Language. It's the standard language for creating web pages. HTML uses a series of simple codes, or "tags," to structure your content and give it meaning.

<!-- This is a comment, it's not displayed on the page -->
<h1>Hello, World!</h1>
<p>This is a paragraph of text.</p>

Pro Tip

HTML provides structure and meaning to your content, making it accessible to both users and search engines.

Basic Page Structure

Every HTML document follows this pattern

Every HTML page has a basic structure. It always starts with a doctype declaration and includes the <html>, <head>, and <body> tags.

<!-- Defines the document type -->
<!DOCTYPE html>
<html>
<head>
    <!-- Contains meta-information like the title and links to stylesheets -->
    <title>My Web Page</title>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
<body>
    <!-- This is the visible content of your page -->
    <h1>My First Heading</h1>
    <p>My first paragraph.</p>
</body>
</html>

HTML Attributes

Adding extra information to elements

Tags can have "attributes" that provide extra information about the element. Attributes are always placed in the opening tag and follow the pattern: name="value".

<!-- The 'src' attribute specifies the image source -->
<img src="image.jpg" alt="A beautiful sunset">

<!-- The 'href' attribute specifies the link destination -->
<a href="https://www.google.com" target="_blank">Visit Google</a>

<!-- The 'class' and 'id' attributes for styling and identification -->
<div class="container" id="main-content">Content here</div>

Important

Always use quotes around attribute values, even for single words. This prevents errors and ensures your code is valid.

Lists: Ordered & Unordered

Organizing information effectively

Lists are essential for organizing information. Use <ul> for unordered (bulleted) lists and <ol> for ordered (numbered) lists. Each list item uses the <li> tag.

Unordered List

<ul>
    <li>HTML Fundamentals</li>
    <li>CSS Styling</li>
    <li>JavaScript Basics</li>
</ul>

Ordered List

<ol>
    <li>Learn HTML structure</li>
    <li>Add CSS styling</li>
    <li>Make it interactive</li>
</ol>

Adding Images and Links

Essential elements for rich web content

Images and links are fundamental to creating engaging websites. The <img> tag displays images, while the <a> tag creates clickable links.

<!-- Image with proper attributes -->
<img src="https://placehold.co/600x400/3b82f6/FFFFFF?text=Awesome+Website"
     alt="A beautiful website screenshot"
     width="600"
     height="400">

<!-- Different types of links -->
<a href="https://www.example.com" target="_blank">External Link</a>
<a href="about.html">Internal Link</a>
<a href="mailto:hello@example.com">Email Link</a>
<a href="#section1">Anchor Link</a>

Accessibility Reminder

Always provide descriptive alt text for images. This helps screen readers and improves SEO. Make your links descriptive too!

Semantic HTML Elements

Modern HTML5 elements with meaning

Semantic elements clearly describe their meaning to both browsers and developers. They improve accessibility, SEO, and code maintainability by providing structure and context to your content.

<!-- Semantic page structure -->
<header>
    <nav>
        <ul>
            <li><a href="#home">Home</a></li>
            <li><a href="#about">About</a></li>
        </ul>
    </nav>
</header>

<main>
    <section id="hero">
        <h1>Welcome to My Website</h1>
        <p>This is the main heading section</p>
    </section>

    <article>
        <h2>Article Title</h2>
        <p>Article content goes here...</p>
    </article>
</main>

<aside>
    <h3>Related Links</h3>
    <ul>
        <li><a href="#">Resource 1</a></li>
        <li><a href="#">Resource 2</a></li>
    </ul>
</aside>

<footer>
    <p>© 2025 My Website. All rights reserved.</p>
</footer>

Best Practice

Use semantic HTML elements instead of generic <div> elements whenever possible. This makes your code more meaningful and accessible.
